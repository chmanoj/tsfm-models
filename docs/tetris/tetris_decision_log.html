Context

TETRIS (TElescoped TRIplets for Irregular Series) is a time series foundation model: one pretrained network that produces zero-shot multivariate forecasts on arbitrary datasets (benchmark target: top GIFT-Eval performance). The model receives only a raw numeric array — any number of channels (covariates split into features and targets), any history length, any sampling frequency, any forecast horizon, no timestamps or metadata — and must predict the targets' future values. It is pretrained once across a heterogeneous corpus of series and applied without per-dataset training. This document is the running log of architecture and training-design decisions for that model, intended to be self-sufficient for an implementation session.

Problem statement

One model over batches of 2-D arrays [T_ctx × C] where some columns are features and some are targets; predict [T_pred × n_targets]. All four dims vary per sample. Hard requirements: (a) no padding-to-max + flatten over covariates; (b) histories longer than the time budget must be used, not truncated, without linear compute blow-up; (c) training must stay torch.compile-compatible despite shape variation in two dimensions (→ static-shape designs preferred, 2-D sequence packing for batching); (d) channels are never dropped or subsampled at task/inference time — when a sample exceeds the token budget, it is always cut in the time dimension (D9 window sampling); training-crop construction may subset channels when manufacturing samples (D13). Hard capacity edge: fitting requires ≥1 token per channel, i.e. C ≤ L_pack − Q; beyond that the buffer length itself must grow.

Decisions

D1Tokenization: span-tagged tripletsdecided

Each token = (t_center, span, variate, content) — a (time, variate, value) triplet generalized so the "time" is an interval and the "value" is an encoding of the window. Tokens of different temporal resolution coexist in one permutation-equivariant set. Token embedding = content-encoding + time-position emb (continuous, relative to forecast origin) + span emb (log₂ patch size) + variate emb (D4) + role tag (feature/target, open O3).

D3Per-tier learned encoders (not shared stats embedder)decided

One input encoder per tier (5 discrete patch sizes ⇒ 5 branches; static shapes preserved). Coarse tiers: small conv/MLP over the raw window. D4Variate identity: random orthonormal IDs (+ content summary)decided

variate_emb(c) = random orthonormal ID (resampled per sample) + content-summary embedding, added at input. The random ID is a register: early layers discover from content which channel matters, later layers route by tag — "discover once, address thereafter." The content embedding is a prior (channel signature → likely behavior); colliding embeddings for identical-content channels is correct behavior, not a bug.

  • Same ID on all of a variate's tokens — every tier and its prediction query tokens (the binding mechanism for decoding).
  • Sample C rows of a random rotation (orthonormal ⇒ guaranteed separation at C=256); resample every training sample so IDs are usable only relationally; fix seed at inference.
  • Content summary: start with stats/spectral feature vector → MLP; ablate a learned pooling encoder over tier-0/1 tokens later.
  • Ablation queued: inject IDs into Q/K only (identity as addressing, not content).
  • Precedents: TabPFN v2 random feature-column embeddings; random-node-feature expressivity results in GNNs; iTransformer content-as-token.

D5Attention: joint causal attention over the (C × T) griddecided

One softmax attention per layer over the full token grid — no separate time-blocks and variate-blocks, no axial factorization. Context attention is causal in time. v1 (training regime, C ≤ 32): dense causal joint attention within each sample — token (c, t) attends to every token (c′, t′) with t′ ≤ t, all channels, all past and same timestep. The only hard masks are sample boundaries, time causality, and the D6 query rules. The log-spaced lag structure survives only as an optional score bias (biases shape logits at zero FLOPs and zero extra tiles). Phase 2: NSA-style learned block selection replaces dense cross-channel reachability for large-C samples and inference (see D8). Horizon query tokens (D6) attend to the entire context and to each other.

  • Direct lagged cross-channel edges: (target, t) → (feature, t−1) is a single hop — solves the lagged white-noise-feature case that defeats axial models by construction.
  • Seasonality without metadata: input is a bare [T × C] array (no timestamps/frequency). Own-row attention covers the entire past, so within-channel seasonality at any period is learnable as content-based QK matching (attention as learned autocorrelation). Under dense attention every cross-channel lag is also reachable — no residual coverage gap.
  • Layer mask schedule: cross-channel edges (key set B) are enabled per layer by a config flag. Early layers run with B masked off — each channel processes only its own history, building clean per-channel representations — and later layers open B to mix channels. This needs no separate block types: it is the same attention op everywhere, with a per-layer mask variant. The depth at which B opens is a hyperparameter (e.g., first ⅓ of layers channel-independent).
  • Cost (dense, within-sample, S = C·N tokens): S²/2 score pairs per layer. Worst training case C=32, N=1280 → S ≈ 41k → ~8.4×10⁸ pairs; typical GIFT-Eval C ≈ 5 → ~2×10⁷. At C ≳ 128 dense explodes (C=256 → ~5×10¹⁰) — rare in training (accepted as a cost spike, bin-packer caps per-sample budget) and the designated job of phase-2 learned selection.
  • Dilution bet: dense assumes the model learns to focus over up to ~40k keys without hard masks (the standard LLM bet), helped by D4 random-ID routing and the optional log-lag score bias. Remedies if it fails, in order: strengthen score bias (see the four-rung ladder in D8/A3) → curriculum via the layer mask schedule → pull NSA forward. Fixed element bands are not a fallback (see D8 utilization data).

D6Decoding & training losses: one-shot masked query tokensdecided

Horizon decoding is non-autoregressive. For a training sample [T+p, C]: context [T, C] tokenized per D1–D3; horizon positions inserted as query tokens — learned [MASK] content embedding + time-position + span + channel ID + role. Queries are inserted only for target channels (known-future feature values, where a dataset has them, enter as real observed tokens over the horizon). Horizon is patchified: each query token decodes a patch (e.g., 32 steps) via a projection head ⇒ p_max = n_query_tokens × patch_size per channel per forward pass. Horizons beyond p_max: chunked rollout (predict p_max, append as context, re-anchor telescope once per chunk — amortizes the D2 origin-reanchoring ~patch-count× vs. per-patch AR).

  • Variable-p training: each sample draws p ∈ [1, p_max]; only ⌈p/patch⌉ queries inserted ⇒ any horizon length ≤ p_max works natively in one pass at inference.
  • Mask directions: queries → all context ✓; queries → queries (same sample) ✓ (trajectory coherence); context → queries ✗ (context representations independent of attached horizon).
  • [MASK] is a learned embedding, not zeros — zero is a valid (and post-normalization, common) data value; zeroed queries would be ambiguous with observed zeros.
  • Point-forecast head, MAE loss (median-optimal; aligns with MASE's absolute-error shape). Quantile/probabilistic head deferred to v2 — GIFT-Eval MASE needs only point forecasts. Note: MASE = model MAE / context seasonal-naive MAE; the denominator is constant w.r.t. predictions (so MAE is the right loss shape) but varies per series ⇒ benchmark-aligned per-series loss weighting is open (O8).
  • Dense auxiliary AR loss: small head on every context token predicting its own channel's next patch in value space (causal context per D5 makes this honest — every position is a forecast origin). Own-channel targets are forced by D4's permutation symmetry: there is no canonical "other channel" to target; cross-channel information still flows into the hidden state via attention — inputs are cross-channel, targets are own-channel. Weight λ ≈ 0.1–0.3. Targets are locally re-anchored increments in step-vol units (see D10) so loss sensitivity is uniform across positions and regimes.

D7Missing data: [NA] state distinct from [MASK]decided

Three content states per token position: observed (tier encoder over values), [MASK] (learned query embedding — "value exists in the future, predict it"), [NA] (learned missing embedding — "no observation here"). The two masks must not be conflated: they carry opposite semantics.

  • Partial missingness within a patch: every tier encoder takes two input channels per timestep — value (NaN→0) and a 0/1 observed-indicator — so a true zero is distinguishable from a filled missing value, and the encoder learns to aggregate around holes.
  • Fully missing patch → [NA] content embedding (+ usual time/span/channel/role embeddings).
  • Grid stays rectangular: dropping missing tokens (allowed in principle by the set formulation) would break column alignment for lag bands and static shapes — rejected. [NA] placeholders preserve both.
  • Loss masking: auxiliary next-patch loss skipped where the target patch is (mostly) missing; horizon loss skipped on NaN ground truth. Same per-position valid side-tensor mechanism as packing's boundary mask.
D8Packed buffer layout: channel-major serializationdecided

Attention kernels (SDPA / FlashAttention / FlexAttention) operate on one sequence axis — the 2-D grid must be serialized; only the order is a design choice. Each sample's grid is laid out channel-major: [ch0: t0..tN | ch1: t0..tN | … | queries], position(c,t) = base + c·N + t. Samples are concatenated as segments in a fixed-length buffer (1-D packing semantics, D-side tensors: sample_id, channel_idx, t_center, span, role, content_state, valid).

Under dense causal attention (D5 v1), attention FLOPs are layout-independent (both orders compute the same causal triangle in time). Channel-major is chosen for everything else: the dense AR aux-loss target (D6) is a shift-by-one gather within a contiguous row; tier-encoder writes and per-channel readouts are stride-free; horizon queries group cleanly; and phase-2 selection blocks align with channel semantics (A4 below).

Hard implementation rule — no buffer-index positional encoding anywhere. The buffer index is a storage address, not a feature: all token geometry must come from side tensors (continuous time embedding on t_center; relative attention biases computed from t_q − t_k; channel identity per D4; span per D1). Attention + per-token FFN are permutation-equivariant given these tags, so serialization order has zero effect on the computation or on learnability (cf. MAE encoding randomly-ordered patch subsets via position embeddings alone; packed LLM pretraining with per-document position reset). Applying stock RoPE / learned absolute embeddings over the packed sequence index — the default in LLM codebases — would scramble semantics under packing and shifting contexts, and is the single most likely implementation bug in this design.

Assumption register — what the layout choice leans on, and what breaks if wrong:

  • A1 — Kernel block granularity B ≈ 128 (FlexAttention BlockMask default; FA2/FA3 tiles 64–128). If wrong (future kernels efficient at B=16–32): time-axis block sparsity regains room even at N/B=10 → re-run the dense-vs-bands tradeoff.
  • A2 — Training C ≤ 32 ≪ B; N ≈ 1280 (N/B ≈ 10). Makes dense affordable and fixed bands pointless (log centers collapse into ~6 of 10 block offsets). If wrong in C (mix shifts to C ≳ 100): dense grows ∝C² — pull phase 2 forward; at C ≥ B, TM block semantics change entirely. If wrong in N (telescope budget grows to 5–10k): dense grows ∝N² — time-axis sparsity (bands or NSA along own-row) returns.
  • A3 — Dilution bet: the model learns to focus over up to ~40k keys without hard masks, helped by D4 ID routing and the optional log-lag score bias. The bias is an additive logit term (a FlexAttention score_mod) computed from side tensors — e.g. b = β·𝟙[same channel] + Σ w_m·φ_m(Δt) with log-spaced lag bumps φ_m; a hard mask is the limit of an infinitely strong bias, so "strength" is a continuous dial. If wrong, escalate the ladder: (1) magnitude — initialize same-channel offset and near-lag bumps large (+2–4 logits) so step-0 attention is near channel-independent/local; learnable coefficients relax where data justifies; (2) structure — per-channel-pair low-rank lag kernel b += Σ α_m(u_{c_q},u_{c_k})·φ_m(Δt), α from D4 content embeddings (precomputed C×C×M table per sample); (3) gating — zero-init learned per-head gate on the cross-channel pathway (ReZero/Flamingo style): exactly channel-independent at init, cross-channel flow admitted under gradient pressure — robust to noisy early QK logits; (4) annealing — start strong, decay the prior over early training (composes with the D5 layer mask schedule); then pull NSA forward. Diagnostics that trigger escalation: per-head attention entropy pinned near log S; own-channel mass stuck at ~1/C or ~1; a synthetic probe with a planted lag-k white-noise feature→target dependency failing to beat the channel-independent baseline. Element bands are not a fallback (table above).
  • A4 — NSA relevance-structure hypothesis (layout-critical): cross-channel relevance is channel-selective (specific driver channels at specific eras / lead-lag structure), not synchronously global. A CM selection block = "channel c′, time-era b" — one selected block is dense in relevant content, composes with D4 route-by-ID, and own-history era selection only exists as a coherent block in CM. If wrong — if relevance is dominated by synchronous events (common shocks where the joint cross-channel snapshot matters): a TM block captures the event in 1 selection slot while CM needs up to C slots — at fixed top-k, CM starves the selection budget. That is the concrete failure mode that would favor time-major or a hybrid.
  • A5 — Reversibility: layout is a buffer permutation, not architecture. Switching CM↔TM, or re-serializing between stages (CM backbone + TM-gathered views for an event-selecting NSA stage), is a gather costing memory bandwidth, zero math. A wrong A4 is a refactor, not a redesign.
  • A6 — One-shot decoding assumed (no KV-cache pressure on layout): under D6's one-shot decoding and D2's origin-anchored telescope, caches are invalidated by re-tokenization, so layout has no caching implications. If decoding ever becomes incremental/streaming: time-major appends new timesteps at the buffer end (stable prefix, cache-reusable) while channel-major inserts into every row (cache dead) — this analysis reopens.

Hedge under A4 (planned escape if both structures matter): the two structures aren't mutually exclusive in reality (markets have both lead-lag drivers and common shocks). If evidence demands both, the design answer is two selection groups — some heads select CM channel-era blocks, others select synchronous windows via a strided/secondary view — rather than a layout flip; NSA's grouped selection accommodates that. Logged as a candidate further update to the model if needed.

D9Packing pipeline: mask truth table, window sampling, streaming packer, cost-bucketed schedulingdecided

1 · Mask truth table. Side tensors per position: s (sample_id), c (channel), t (time), role ∈ {ctx, qry}. Attention q→k allowed iff:

s[q]==s[k] AND ( (role[k]==ctx AND (role[q]==qry OR t[k]<=t[q])) OR (role[k]==qry AND role[q]==qry) )

ctx→ctx: time-causal, channel-blind (dense, ≤ includes same-time and self) · qry→ctx: always · qry→qry: always (trajectory coherence, bidirectional — query tokens carry no input information, so no causality needed) · ctx→qry: never (context representations independent of attached horizon). Pad positions get s = −1 plus a q==k self-attend exception (NaN guard; outputs discarded). BlockMask built once per batch on GPU and reused across all layers (mask is depth-invariant in v1).

2 · Window sampling (per raw series [T_raw, C]): sample horizon p (variable-p, D6) → Q = n_targets·⌈p/patch⌉; per-channel token allowance n = ⌊(L_pack − Q)/C⌋; raw coverage T_cov = telescope-coverage(n) (invert the D2 ladder); forecast origin sampled uniformly over valid positions; context = up to T_cov steps ending at the origin. Long series yield multiple crops per epoch across all their regimes. Segment size S = C·n_used + Q is known from the spec alone (no tokenization needed to pack). Consequence recorded: channel count and temporal depth trade off by construction under a fixed buffer (C=128 ⇒ shallow context); deep telescopes are trained via solo buffers (univariate / low-C samples can fill L_pack alone).

3 · Streaming packer. Source mixture (weighted) → window sampler emits segment specs → reservoir of K ≈ 1000 specs (doubles as shuffle buffer) → best-fit decreasing: repeatedly place the largest spec that fits the open buffer's remaining length; emit when nothing fits or residual < tolerance; pad tail (s=−1); refill reservoir. Length is the only packing constraint. Large samples occupy solo buffers; packing exists purely to stop small-sample padding waste (expected tail waste: low single-digit %). CPU workers tokenize and assemble chosen specs; every buffer is exactly [L_pack] + side tensors ⇒ one compile graph, composition varies as data.

4 · Cost-bucketed step scheduling. Buffer cost = Σᵢ S_i²/2 attention pairs, known from specs. Per-token attention ≈ 4d·S vs FFN ≈ 12d² ⇒ parity at S ≈ 3d (≈768 tokens @ d=256); a solo 64k buffer ≈ 2×10⁹ pairs vs ~3×10⁷ for a packed-small buffer — ~40–60× spread at identical shape. Remedy: a scheduler window (64–256 buffers) sorts by cost and forms each global step from similar-cost buffers across ranks — giants travel together in occasional slow steps, no straggler tax, shapes unchanged. Caveat recorded: solo-buffer steps carry few samples (sometimes one) ⇒ keep giant-step frequency in mind when weighting the data mixture (a full-depth multichannel series has outsized per-occurrence gradient influence).

5 · Loss gathers. Aux next-patch target of position p = raw patch behind p+1; valid iff s[p+1]==s[p] AND c[p+1]==c[p] AND span[p+1]==span[p] (skip ~4 tier-boundary positions per row) AND target patch sufficiently observed (D7). Horizon readout = precomputed index_select over query positions; predictions scatter to (s, c, horizon step); NaN ground truth dropped via valid bits.

D10Normalization: per-channel anchored-arcsinh (reversible), locally re-anchored lossesdecided

Input representation (per channel, per sample, context statistics only): z = arcsinh((x − a)/σ_Δ) with a = median(last 32 raw steps) (anchor window 8–16 is a tuning point for steep trends — the last-32 median visibly lags) and σ_Δ = 1.4826·median|Δx| with fallback chain: if 0 → mean|Δx| (intermittent/quantized) → level-IQR/1.349 → 1 (genuinely constant series degrade gracefully to all-zeros); plus relative floor σ_Δ ≥ 10⁻³·IQR/1.349. Constants 1.4826 and 1.349 are Gaussian consistency factors (MAD = 0.6745σ; IQR = 1.349σ) so all robust scales estimate the same σ. Exact inversion x = sinh(z)·σ_Δ + a denormalizes horizon predictions; no leakage. Normalize first, then aggregate coarse tiers (per D3); coarse tiers carry their own per-tier σ_Δ (robust median |Δ| of the tier-aggregated sequence).

Rationale (verified by stress test, 8 synthetic series types): normalization allocates a finite resolution budget along the series; z-score/RevIN spends it at the context mean level — for trending series, the middle of history — leaving the forecast-relevant recent dynamics in a sliver (regime-jump case: recent window occupies 1% of the representation range; loss sensitivity to a one-step-vol error at the horizon: 0.0015 vs 0.139 anchored — up to ~90× difference; 13× on GBM trend, 5.8× on random walks). Anchored-arcsinh spends resolution at the current level and the horizon, log-compressing the past gradually (slope 1/|z|) — the value-space twin of the telescope. |z| > 2 in old context is by design, not a defect: arcsinh stays monotone and exactly invertible everywhere.

Known weaknesses (recorded): (a) no advantage on stationary series (schemes equivalent there); (b) less fine old-history detail than z-score on trending series (mitigated by D2's telescope coarsening old history anyway); (c) smooth seasonal series with amplitude ≫ step-vol get peak compression (|z|≈4 at cycle tops) — mitigation if needed: blended scale s = max(σ_Δ, λ·IQR of last cycle), ablatable.

Losses are locally re-anchored (fixes a real interaction with D6): in global z-space, old positions on non-stationary series sit where slope ≈ 1/|z| is tiny, making the dense AR aux loss blind there — the "every position is a forecast origin" property would silently collapse to recent origins. Fix: every prediction target — aux and horizon — is movement from the prediction point's local level in step-vol units, arcsinh-compressed: aux target at position t = arcsinh((x_{t+1..t+P} − level_t)/σ_Δ^{tier}); horizon target = arcsinh((y − a)/σ_Δ) (identical form; a is the origin's local level). Uniform loss sensitivity at every position in every regime; global anchored z remains the input representation. Loss space: normalized (equal per-sample influence); arcsinh-space MAE default, vol-unit-space (sinh-inverted) as ablation — arcsinh-space down-weights large moves which MASE does not forgive.

Cross-channel scale receipts: per-channel normalization destroys real relationships ("channel A is 100× channel B"); re-inject per-channel summary features (log σ_Δ, log|a|-relations, fallback-tier flag) into the D4 content embedding.

Experiment hedge (mandated) — all schemes behind one-line config toggles. Implement normalization as two independent config axes so any combination is a single-line training-config change, no code edits: input_norm ∈ {anchored_arcsinh (default), zscore_arcsinh} and loss_target ∈ {locally_reanchored (default), global_norm_space}, plus loss_space ∈ {arcsinh (default), vol_units}. All combinations must run. Rationale for the toggle requirement: the locally re-anchored targets are visually noise-dominated (most of a 1-step increment is irreducible — true of next-token LLM losses too, and patch-level P=32 targets have better signal-to-noise), but if they prove hard to learn in practice the fallback to global-space loss targets must cost one line. Decide by validation MASE on the real mixture; anchored vs z-score input is the headline ablation (z-score = field-standard control). Synergy noted: σ_Δ scaling makes per-sample normalized MAE ≈ MASE-aligned (the MASE denominator is the naive forecaster's step error) — partially resolves O8.

D11Feature/target roles: embedding, no hard masks, light role augmentationdecided
  • Role embedding: one learned embedding, role ∈ {target, feature}, added at input on every token. Two states suffice — "known-future feature" (KFF) is distinguished by its time embedding (t > origin), not a third role.
  • No role-based hard masking: asymmetric schemes (features attend only features, etc.) contradict the D5 dense philosophy and save no meaningful FLOPs. If role asymmetry matters, it enters as a role-pair score bias b(role_q, role_k) — four learned scalars per head, a natural extra rung on the D8/A3 ladder.
  • KFF tokens need zero new machinery: a KFF is an observed (ctx-role) token at a horizon timestamp. The D9 truth table already handles it: context can't attend it (causality), queries attend it (qry→ctx), it attends its causal past, never attends queries. Queries reading KFF at the same horizon timestamp (Δ=0 cross-channel edge) implement cross-sectional regression — e.g., forecasting one treasury tenor from other tenors at the same date, where features are unpredictable from their own past but informative contemporaneously (the use case that motivated direct cross-channel edges in D5).
  • Aux loss applies to all channels including features and KFF horizon tokens: own-dynamics supervision is what teaches the model to read features; unpredictable features simply yield noise-floor aux terms (harmless, informative about unpredictability).
  • Role designation policy: (1) datasets with native role labels (GIFT-Eval past/future feature columns; custom feature-dependent datasets) pass through as labeled; (2) all-target multivariate datasets (the GIFT-Eval majority) stay all-target by default; (3) light role augmentation so the feature role isn't undertrained: with crop-level probability q ≈ 0.2 (tunable), demote each channel to feature w.p. ~0.5 keeping ≥1 target; each demoted feature reveals its horizon (becomes KFF) w.p. ~0.5. Most data stays as intended — augmentation exists only to train the role machinery and manufacture feature→target supervision from unlabeled data (per D4's data requirement). Exact q / demotion / reveal rates are O6 mixture tuning points.
  • Custom feature-dependent datasets (cross-sectional structure: rate curves, lead-lag pairs) are an explicit O6 data-construction item — they are the primary training signal for the KFF path and D4 routing.
D12Telescope schedule: patch vocabulary, allocation policy, design point, context expansiondecided

Patch-size vocabulary (fixed): {4, 8, 16, 64, 256, 512} — six per-tier encoders (D3), six per-tier σ_Δ statistics per channel (D10). The vocabulary is fixed so encoders are shared across all samples; only per-tier token counts vary (counts are data, not shape).

Design point (C=21, L_pack=1024): counts [8, 8, 8, 8, 8, 4] = 44 tokens/channel → coverage 32+64+128+512+2048+2048 = 4,832 steps/channel; context 924 tokens + 84 query tokens (output patch 16, 4 queries/channel) = 1,008 ≤ 1,024. Resolution profile: last 32 steps at patch-4, ~220 steps at ≤16. Single-pass horizon at full C=21: 64 steps/channel; longer horizons via iterative 64-step chunks (predict → append → re-anchor telescope → repeat; ~12 passes for a 720-step horizon — acceptable: only 15/97 GIFT-Eval datasets exceed 720, point forecasts only). These are the C=21 instances of the general D9 budget formula — a univariate sample at the same L_pack gets both deeper context and a much longer single-pass horizon.

Allocation policy (parametric in n tokens/channel and T_raw): halving prior (finest tier gets ~half the tokens, halving down the ladder) → compute coverage → rebalance: coverage > T_raw ⇒ walk tokens coarsest→finest until coverage ≈ T_raw (short series become mostly patch-4); coverage short even with coarse tiers maxed ⇒ the D9 window sampler has already cropped. Deterministic, integer ops per sample.

Context expansion 1024 → 2048: double per-tier counts to [16, 16, 16, 16, 16, 8] (coverage 9,664/channel at C=21). Works because: zero new parameters (same vocabulary ⇒ same encoders/span embeddings); no positional territory to extend (D8 no-buffer-index rule; t_center ages are already trained by solo univariate buffers reaching tens of thousands of steps at L=1024). What shifts is adaptation-scale only: more keys per query, doubled worst-case attention (D9 scheduler absorbs), changed count distributions ⇒ short fine-tune from the 1024 checkpoint. Alternative expansion axes recorded: (a) asymmetric reallocation — spend new budget on finer recency at same coverage (empirical fork, leave open); (b) vocabulary extension with a new coarser patch (×2048/×4096) for more coverage — adds one fresh encoder needing brief training, the more expensive lever.

D13Data strategy: GIFT-Eval-targeted corpus, two-phase schedule, test-tracked trainingdecided

Corpus: (A) GIFT-Eval Pretrain — 88 datasets (71 univariate, 17 multivariate), ~4.5M series, ~230B points, 13 frequencies, non-leaking backbone; (B) the train split of GIFT-Eval TrainTest (~144k series) — legal in-distribution gold, used by current leaderboard models (Moirai 2.0 precedent); (C) Mystic-B multivariate KernelSynth — variates built from 3–4 kernels sampled out of a shared 7-kernel bank, so channels correlate through latent factors (the factor-model prior as data; primary cross-channel training signal; knobs: bank size, kernels/variate, overlap degree, C distribution); (D) optional filler: univariate KernelSynth + Chronos-Mixup. Lead-lag covariate datasets explicitly deferred — the architecture is capable (D5/D11), GIFT-Eval doesn't need them.

Mixture weights: effective weight = manual multiplier × size^α (α ≈ 0.3–0.5, per-dataset caps; never proportional-to-size). Config map dataset_weights: {id: multiplier}; multiplier 0 removes a dataset and runtime renormalization over the surviving set redistributes its mass automatically (also covers locally-unavailable datasets). Synthetic share ~10–15% total, config-exposed.

Two-phase schedule, single run, restartable: Phase 1 — broad pretrain on A+C+D with B at natural weight. Phase 2 (last ~10–20% of steps, phase2_start in config) — anneal: upweight B and switch the crop sampler to test-matched marginals derived automatically from the 97-config table (horizon set incl. ×10/×15 extensions, context lengths, channel counts, frequency marginals; phase2_crop_distribution: auto_from_test_configs, manual override allowed). Checkpoint discipline: periodic checkpoints + mandatory end-of-phase-1 checkpoint (model + optimizer + dataloader state); phase 2 is always re-runnable from it with a tweaked config and a re-warmed LR — iterating the anneal costs 10–20% of a run, not 100%.

Crop/shape distribution (D9 sampler parameters): log-uniform horizon p within each sample's budget (phase-2 reweighted to test horizons); random origins; deliberate long-crop sampling from long series so coarse tiers and deep t_center ranges train (the D9 giant-step frequency knob lives here); temperature-balanced frequencies so rare ones aren't starved.

Multivariate construction: native multivariate datasets cropped as random channel subsets of 2–21 — this is training-crop manufacturing and is explicitly in scope (the "never drop channels" rule of D9 governs task serving/inference, where the task defines C and only time may be cut; constructing training samples from a 370-channel source by subsetting is legal and necessary). Plus, at low rates: related bundles (timestamp-aligned series from the same univariate dataset — genuine cross-sectional correlation: traffic sensors, electricity clients) and unrelated bundles (cross-dataset channel salads — dilution-robustness training: learn to ignore useless channels). D11 role augmentation (q ≈ 0.2) applies on top.

Hygiene: dedup across A and B (LOTSA-descended overlap risk); min-length filter; NaN-fraction cap (~90%); constant series kept (D10 degrades gracefully and they occur in test).

Monitoring: train-split validation windows are the primary signal — all architecture and ablation decisions (normalization toggles, mixture weights, A3 ladder escalation) are made on them. The GIFT-Eval test split is tracked for record only, never for decisions: periodic test loss and true-formula MASE on a deterministic shard of the first 100 windows per config (shrinkable for speed) — cheap because the seasonal-naive denominator is context-only and precomputed once per series; full official GluonTS harness at every saved checkpoint as ground truth (reconcile the in-training proxy against it once).

D14torch.compile mechanics: one static graph, FlexAttention, grouped tier dispatchdecided
  • One static graph by construction: every batch is [B, L_pack=1024] tokens + fixed-shape side tensors; all per-sample variability (segment count, tier counts, query positions) is tensor contents, never shapes — no mark_dynamic, no recompiles. Packing, reservoir, window sampling, allocation, normalization statistics, and crop-time weights run on CPU dataloader workers, outside the graph. Compile the full training step. The 1024→2048 expansion (D12) is one fresh compile at a new static shape, not dynamism.
  • Attention: FlexAttention from the start — D9 truth table as mask_mod (BlockMask built once per batch on GPU; at S=1024 / 8×8 blocks the construction cost is noise), with score_mod as the ready socket for the A3 bias ladder and the phase-2 NSA migration path. Rejected alternative: plain SDPA with a materialized [1024×1024] bool mask per buffer — works at this scale but forces a kernel migration the moment biases or learned sparsity activate.
  • Per-tier encoder dispatch (MoE-style grouped): six encoders with different raw input widths (4…512) cannot be one op, and data-dependent Python branching would graph-break. At pack time the CPU emits per-tier index tensors with static capacities (cap_k per buffer from the D12 allocation bounds + headroom, padded, asserted); in-graph: gather → [cap_k, patch_k] encode → scatter into [B, 1024, d]. Six fixed-shape gathers/scatters, fully compileable, negligible padding waste. Rejected: pad every patch to 512 for one span-conditioned encoder — ~128× memory on fine tiers, contradicts D3.
  • Loss gathers: query readout and aux-target indices precomputed per batch as fixed-size padded index tensors + valid masks; in-graph gather/index_select only.
D15Seasonal-naive loss weightingdeferred (v2)

Decision: postpone — no further loss complexity in v1. Recorded proposal for when it's picked up: multiply each sample's loss by w = σ_Δ / (seasonal-naive MAE on context at the ACF-detected period), precomputed at crop time in the CPU workers (one ACF + one naive pass), behind a config toggle loss_weighting ∈ {none (default), seasonal_naive}, decided by train-split validation MASE. Rationale for deferral: D10's σ_Δ scaling already aligns per-sample loss with naive-1 MASE; the residual gap to the seasonal denominator is small; period detection is noisy on short contexts (added variance); and test MASE is directly tracked anyway (D13).
