How one batch flows through the model

A step-by-step trace with every tensor and its exact shape, worked through concrete example samples. Where a shape uses a variable, the variable is defined and its value for our examples is given. Follows decisions D1–D15.

tiers6
Nesting: slot (one of L token positions) ⊂ buffer (one packed row of L slots, holds one-or-more whole samples + pad — the GPU's "sequence") ⊂ batch (B buffers per step).
Batch tensor after embedding is [B, L, D] = [4, 1024, 256]. A buffer is not one sample.
Types used in the signatures below. Tensors written as name: dtype[shape]; structs are plain dataclasses.
LoaderItem = tuple[Tensor float32[C, t_i], int n_features, int n_targets]
SegmentSpec = dataclass{ C: int, n_features: int, n_targets: int, origin: int, p: int, channel_roles: ChannelRole[C], Q: int, K: int, Q_total: int, n: int, counts: int[6] } — scalar ints (origin = index into the raw series where context ends / horizon begins), plus the per-channel channel_roles list (each ∈ {FEATURE, TARGET, KFF}; KFF = a feature whose horizon is revealed) and the 6-int per-tier counts. This is CPU pack-time data — the variable-length list never reaches the GPU, so it does not affect the static compiled shapes.
TokenMeta = dataclass{ tier_id, channel, raw_start, variate_uid: int; role: Role{CTX,QRY}; content_state: ContentState{OBSERVED,MASK,NA}; role_ft: RoleFT{FEATURE,TARGET}; t_center: float; valid_aux: bool } — one per token. The three enums are independent (D6/D7/D11): role drives the mask, content_state selects the content slot, role_ft is the added role embedding. A known-future feature (KFF) is not a separate role — it is an observed CTX feature token at t_center > 0 (distinguished by time, D11).
Stats = dataclass{ a: float; sigma_delta: float[6] } — one anchor and one per-tier scale per (sample, channel), constant across all timesteps (not rolling/per-step; this is what makes inversion possible with two numbers).
Batch = the collated NamedTuple of GPU tensors (full field list in Stage 4).

Telescope allocation

in: spec: SegmentSpec, t_i: int
out: spec (counts filled), meta: list[TokenMeta]

allocate(spec: SegmentSpec, t_i: int) → tuple[SegmentSpec, list[TokenMeta]]. Fills spec.counts[6] and emits one TokenMeta per token of this segment (length = ctx_tokens + Q_total). Splits n across 6 tiers by the ratio prior (front-loaded ≈ [8,8,8,8,8,4]), then trims coarse→fine until coverage ≤ t_i.

3

Normalization (anchored arcsinh)

in: raw: float32[C, ctx_len]
out: norm: float32[R], obs: bool[R], stats: Stats

normalize(raw: float32[C, ctx_len]) → tuple[float32[R], bool[R], Stats], per channel, context-only, exactly invertible. z = arcsinh((x−a)/σΔ), a = median(last 32), σΔ = 1.4826·median|Δx| (fallback chain & per-tier σΔ).

outputshapenotes
norm_values[R] float32R = total normalized raw steps the buffer's tokens reference (per-buffer, large for deep crops). Indexed by each token's raw_start.
observed_mask[R] boolNaN→0 + indicator (D7).
a, σΔ[n_samples, C], [n_samples, C, 6]one anchor a per (sample, channel) (the level at the origin); one σΔ per (sample, channel, tier). Constant across all timesteps — the whole context+horizon of a channel is normalized by the same a, σΔ (not a rolling per-step normalization). For inversion at Stage 9 & as scale receipts to the variate embedding (D4).

R is the only quantity not bounded by L — it's the raw history length, e.g. Sample B references ~111k steps. The L-bounded tokens carry pointers (raw_start) into this store, not the values themselves. Note the shape contrast: norm_values is per-timestep [R] (the actual normalized series), but a/σΔ are just a handful of scalars per channel — because anchored-arcsinh applies one (a, σΔ) uniformly to the whole channel, it is not a per-timestep transform.

4

Pack — segments into buffers

in: list[(spec, meta, norm, obs, stats)]
out: batch: Batch

collate(items: list[tuple[SegmentSpec, list[TokenMeta], float32[R], bool[R], Stats]]) → Batch. Stateless. Lays each sample channel-major into a buffer, best-fit-packing small samples together, tail padded with s=−1. The batch is B=4 buffers:

5

FlexAttention mask — one L×L per buffer

in: batch: Batch
out: block_mask: BlockMask[B,L,L]

Attention never crosses buffers — each of the B buffers gets its own independent L×L = 1024×1024 mask; the batched BlockMask is [B, L, L]. Within a buffer, a query slot q attends key slot k iff they're the same sample AND the causal/role rule holds:

sample_id[q]==sample_id[k] AND ( (role[k]==CTX AND (role[q]==QRY OR t[k]≤t[q])) OR (role[k]==QRY AND role[q]==QRY) )

D9 truth table (role ∈ {CTX, QRY}). A KFF token is role=CTX, so it falls out for free: queries read it (qry→ctx), context can't see it (causality, t_k>t_q), it attends its own causal past, and it never attends queries (D11). No KFF special-casing in the mask.

own-channel causal cross-channel causal (same sample, t_k≤t_q) query→context masked / skipped

Two buffers shown — they are separate matrices, not one shared space. Within buf 2, the three packed samples (C, F, E) form their own block-diagonal sub-blocks; cross-sample pairs inside the same buffer are masked too.

Depth-invariant: one BlockMask per batch, reused by every transformer layer. Built once on GPU. FlexAttention from the start so the A3 score-bias ladder and phase-2 NSA plug into score_mod with no kernel change.

7

Per-tier encoder dispatch → content

in: norm_values: f32[B,R], tier_id/raw_start: [B,L]
out: content: float32[B,L,D]

Goal: fill the content vector [B, L, D] = [4,1024,256] — one D-dim vector per token. Six encoders (one per patch width) can't be a single op, so route by index. To stay compile-static, each encoder runs at a fixed capacity ENCODER_CAP (config, default = L = 1024), not the dynamic per-batch count — unused rows are sentinel-padded and masked after. Walk through it with concrete shapes:

The 2 in [CAP, Pₖ, 2]: each timestep fed to an encoder carries two numbers — channel 0 = normalized value, channel 1 = observed-indicator (1 if real, 0 if it was NaN/missing and filled, per D7). This lets the encoder tell a genuine 0 from a filled-missing 0.
stepoperationshape
0 · flattentreat the batch as B·L = 4096 token slots[4096] of (tier_id, raw_start, content_state)
1 · routefor tier k, a fixed-length index list of slots with content_state==OBSERVED & tier_id==k (context + KFF; real count Mₖ ≤ CAP; padded with sentinel to CAP)idxₖ : int[CAP] (CAP=1024)
2 · gather windowspull each routed token's Pₖ-wide (value, observed) window from norm_valueswinₖ : [CAP, Pₖ, 2]
3 · encodeencoderₖ : (Pₖ·2 → D) maps each window to one D-vectorencₖ : [CAP, 256]
4 · scatterwrite the Mₖ real rows back to their slots; sentinel rows discardedcontent_flat : [4096, 256]
5 · reshape→ batch formcontent : [4, 1024, 256]
Why CAP = L and not a tuned per-tier cap. Each encoderₖ runs [CAP, Pₖ, 2]→[CAP,256] at fixed CAP, so the graph compiles once with no per-tier numbers to invent and no risk of dropping tokens. Cost of padding every tier to 1024 vs. routing the real counts: encoder FLOPs rise from ~0.9 to ~6.8 GFLOP/batch — but that is 6.6% vs 0.9% of the ~103 GFLOP transformer backbone, i.e. ~6% extra total compute, and ~7 MB/buffer of gather transient. Accepted for simplicity. ENCODER_CAP is a config value (default L) so it can be lowered with an assert if L=2048 or large batch makes the tier-5 transient pinch. Note: pad the token-count dimension only — keep each encoder's native Pₖ width (tier-0 stays [CAP,4,2], never [CAP,512,2]).

8

Transformer stack ×N

in: tokens: f32[B,L,D], block_mask
out: hidden: float32[B,L,D]

Pre-norm blocks (attention + MLP), each using the Stage-5 BlockMask [4,1024,1024]. Shape is invariant: [4,1024,256] → [4,1024,256]. No positional encoding added here — position is already in the token embedding.

9

Heads & loss

in: hidden: f32[B,L,D], batch
out: loss: float32[] (scalar)
pathhead runs onheadoutput shape
horizonall L slots (dense, static)D→16[B,L,16]
aux next-patchall L slots × 6 tier heads (dense, static)6 heads D→Pₖ[B,L,Pₖ] ×6
  • Horizon (option β — dense, no gather): run the head over the full [B,L,256]→[B,L,16], then take the masked mean against horizon_target using target_valid (true only at query slots). Loss = (|pred−tgt|·valid).sum()/valid.sum() — fixed-shape tensors, the selection happens in the reduction, so no dynamic shape and one compile. (~5× wasted matmul on a tiny 256→16 head — negligible.)
  • Aux (also dense): run all six tier heads over the full [B,L,256]→[B,L,Pₖ]; each token then picks its own tier's output via tier_id, compared to the collator's raw-time target, masked by valid_aux. Runs 6 heads where 1 would do — accepted, the heads are tiny. Per-tier weight vector (coarse tiers down-weightable).
  • Total: L = horizon_MAE + λ·Σₖ wₖ·aux_MAEₖ, λ≈0.1–0.3.
Inference replaces loss with: gather query outputs → invert normalization x=sinh(z)·σΔ+a → if horizon > single-pass budget, append & re-anchor & repeat (D6 chunked rollout).
